## الهدف

تحسين ملف PDF النهائي (شهادة المعادلة) ليعرض بوضوح قرار كل مادة مع ملاحظات المشرف وسبب الرفض، وإصلاح حجم/شكل اللوقو، وإرفاق الرسالة الأخيرة من المشرف.

## التغييرات المطلوبة

### 1) إصلاح اللوقو في رأس الـ PDF
- **المشكلة الحالية:** اللوقو يُرسم في مربع 70×70 بينما أبعاده الأصلية 666×375 (نسبة 16:9 تقريبًا). هذا يشوّه شكله.
- **الحل في `src/lib/pdfHelpers.ts` داخل `drawBrandedHeader`:**
  - الحفاظ على نسبة العرض إلى الارتفاع: عرض 96pt × ارتفاع ≈ 54pt.
  - رفع ارتفاع شريط الرأس من 110 إلى 120 ليتناسب مع اللوقو الأكبر.
  - توسيط اللوقو رأسيًا داخل الشريط (y ≈ 33).
  - إعادة محاذاة بلوك نصوص الجامعة بجانب اللوقو ليبدو نظيفًا.

### 2) إعادة تصميم بلوك "المادة" داخل PDF القرار النهائي
في `src/lib/exportDecisionPdf.ts` (وضع الدفعة وكذلك المادة الواحدة) يصبح كل بلوك مادة يحتوي على:

1. **اسم المادة** (Source course)
2. **شرح/ملخص المادة** (Course description) — يُؤخذ من `saudi_course_description` للمادة الواحدة، أو من `summary` لكل مادة في الدفعة.
3. **القرار:** APPROVED / REJECTED مع شارة ملوّنة بارزة.
4. **سبب الرفض / الملاحظات:** نعرضه فقط عند `rejected` كصندوق أحمر فاتح بعنوان "Reason for rejection". عند `approved` نعرض ملاحظات الاعتماد بصندوق أخضر فاتح بعنوان "Approval notes" — وفقط إذا وُجدت.
5. **حذف الحقول غير الأساسية** من بطاقة المادة في الدفعة (مثل سطر AUT equivalent المختصر) واستبدالها بشكل أوضح: عنوان المادة → الوصف → القرار → السبب/الملاحظة.

نحافظ كذلك على:
- ترويسة الجامعة الرسمية + اللوقو + رقم الطلب + الختم + توقيع المشرف + التذييل (الأساسيات الموجودة الآن).

### 3) صفحة "رسالة المشرف الأخيرة" المرفقة
في نهاية ملف الـ PDF نضيف **صفحة جديدة** بعنوان "Final supervisor message":
- صندوق رسمي يحتوي:
  - اسم المشرف (`reviewer_name`).
  - تاريخ القرار.
  - النص الكامل من حقل `admin_notes` (الموجود حاليًا في الواجهة كرسالة المشرف الأخيرة عند الاعتماد/الرفض).
  - في حالة الدفعة: قائمة منسّقة لكل مادة مع قرارها وملاحظتها الفردية إن وُجدت في `decision.notes`.
- توقيع وختم في أسفل الصفحة.

### 4) تمرير الحقول من صفحة الإدارة إلى الـ PDF
في `src/pages/Admin.tsx` داخل `printPdf`:
- التأكد من تمرير `admin_notes` و `reviewer_name` و `reviewed_at` (مُمررة بالفعل).
- تمرير `saudi_course_description` لكل مادة في الدفعة (موجود في `BatchCourse.summary` — سنستخدمه مع fallback نصي إنجليزي مأمون).

## ملف الـ PDF بعد التعديل سيحتوي على:

```text
┌──────────────────────────────────────────┐
│ [LOGO 96×54]  Aqaba University of Tech.  │  ← لوقو بالنسبة الصحيحة
│              Faculty of IT — Committee    │
├──────────────────────────────────────────┤
│ Course Equivalency Certificate           │
│ APPROVED / REJECTED / MIXED   #ABCDEF12  │
├──────────────────────────────────────────┤
│ Student Information (الاسم/الإيميل/...)   │
├──────────────────────────────────────────┤
│ Committee Decisions (N courses)          │
│  ┌────────────────────────────────────┐  │
│  │ Course #1 — APPROVED               │  │
│  │ Name: ...                          │  │
│  │ Description: ...                   │  │
│  │ ✅ Approval notes: ...             │  │
│  └────────────────────────────────────┘  │
│  ┌────────────────────────────────────┐  │
│  │ Course #2 — REJECTED               │  │
│  │ Name: ...                          │  │
│  │ Description: ...                   │  │
│  │ ❌ Reason for rejection: ...        │  │
│  └────────────────────────────────────┘  │
├──────────────────────────────────────────┤
│ Approved by: Dr. ...        [SEAL]       │
├──────────────────────────────────────────┤
│ === Page 2 ===                           │
│ Final Supervisor Message                 │
│  Dr. ... — date                          │
│  "نص الملاحظة الأخيرة كاملة"               │
│  Per-course notes (للدفعة)               │
└──────────────────────────────────────────┘
```

## الملفات المعدّلة
- `src/lib/pdfHelpers.ts` — تكبير اللوقو + الحفاظ على النسبة + رفع ارتفاع الترويسة.
- `src/lib/exportDecisionPdf.ts` — إعادة تصميم بطاقات المواد + صفحة رسالة المشرف الأخيرة.
- `src/pages/Admin.tsx` — تمرير وصف المادة بشكل سليم لكل عنصر دفعة (تعديل صغير في `printPdf`).

## ما يبقى كما هو
- الواجهة بالعربية كاملة، التبديل عربي/إنجليزي.
- نص الـ PDF بالإنجليزية لتجنّب مشاكل الخطوط (المعتمد سابقًا).
- منطق قرارات المواد الفردية وحالة الطلب الكلية في صفحة الإدارة.
- ملف PDF التمهيدي (`exportPreliminaryPdf`) لا يتغيّر.
